# Kickoff Packet

objective:
Define the next smallest useful build target for captains-prediction-companion using the new operator system.

constraints:
- do not break the current app
- keep operator files separate from product code
- prefer one small shippable step
- require file-level proof

scope_in:
- inspect current repo architecture
- identify the most valuable next build target
- define one exact next implementation step

scope_out:
- full rewrite
- multi-agent automation
- auto-trigger routing
- UI redesign unless it is the smallest valuable step

acceptance_criteria:
- one clear next target is selected
- reasons are grounded in the current repo structure
- proof requirements are stated
- implementation scope is narrow and testable

failure_conditions:
- vague roadmap language
- multiple competing next steps
- no proof requirement
- scope drift into a large redesign

required_proof:
- referenced repo paths
- files likely to be touched
- exact next implementation step
- success check for that step

exact_next_step:
Review AGENTS.md, ARCHITECTURE.md, README.md, and the top-level frontend/src structure, then choose the next smallest valuable implementation target.
