# controller-scope-gate

## Purpose
Turn a rough project request into a scoped execution packet for this repo.

## Use when
- a request is vague
- multiple directions are possible
- the next step is unclear
- proof requirements need to be stated before work starts

## Required output
objective:
constraints:
scope_in:
scope_out:
acceptance_criteria:
failure_conditions:
required_proof:
exact_next_step:

## Rules
- one exploration round only
- force convergence after scope is clear
- prefer the smallest useful step
- reject vague completion claims
- require proof before marking work complete
